# Release 2026-05-18

## Summary

This release focuses on product usability and operator support improvements, with multilingual response support, contextual follow-up suggestions, and conversation export.

## Added

- Multilingual chat responses with language-aware request field (`responseLanguage`) and validation.
- Language selector in the chat composer (`English`, `Spanish`, `Portuguese (Brazil)`, `French`).
- Contextual follow-up suggestions generated from answer evidence and role context.
- Conversation export as markdown with transcript timestamps and cited evidence.
- One-command diagnostics scripts:
  - `scripts/diagnostics.sh`
  - `scripts/diagnostics.ps1`

## Changed

- OpenAI chat model configuration updated to `gpt-5.4-mini` across backend, frontend, and tests.
- Structured response mapping now returns follow-up suggestions for both successful and no-context responses.

## Validation

- Backend targeted tests: `ChatControllerTests`, `FallbackRetrievalChatServiceTests`, `ChatResponseMapperTests`.
- Frontend tests: `ChatPage.test.tsx`.
- Backend and frontend builds complete successfully.

## Operational Notes

- Diagnostics script records endpoint connectivity errors in output files when services are down, while still generating artifacts.
- Existing warning-level dependency notices for OpenTelemetry packages remain and should be addressed in a dedicated dependency update cycle.
