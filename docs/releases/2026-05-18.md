# Release 2026-05-18

## Summary

This release consolidates a major platform increment across chat UX, ingestion workflows, and operational reliability. The focus is stronger evidence traceability, safer ingestion/versioning, and better runtime resilience for both local and containerized runs.

## Added

- Streaming chat endpoint (`POST /api/v1/chat/stream`) with progressive frontend transcript updates.
- Rich citation handling in UI and API, including document version and checksum traceability metadata.
- Multi-knowledge-base support (`knowledgeBaseId`) for isolated ingest/retrieval scopes.
- Ingest upload and preview flows for local `.md`, `.txt`, `.pdf`, and `.docx` inputs.
- One-command diagnostics scripts:
  - `scripts/diagnostics.sh`
  - `scripts/diagnostics.ps1`

## Changed

- Ingest now performs checksum-based duplicate detection and returns `409 conflict` for already ingested document versions.
- Forced reingest now applies incremental chunk updates/deletes with embedding reuse for unchanged content hashes.
- OpenAI integration now includes configurable circuit-breaker behavior in addition to retry/timeout policies.
- Structured response mapping includes confidence and usage metadata to support UI evidence and operational reporting.

## Validation

- Backend targeted tests: chat, ingest, and retrieval service suites (`ChatControllerTests`, `IngestControllerTests`, `OpenAIRetrievalChatServiceTests`, `ChatResponseMapperTests`).
- Frontend tests: page/component suites covering transcript, citations, and composer behavior.
- Backend and frontend builds complete successfully.

## Operational Notes

- Diagnostics script records endpoint connectivity errors in output files when services are down, while still generating artifacts.
- Existing warning-level dependency notices for OpenTelemetry packages remain and should be addressed in a dedicated dependency update cycle.
